# 健康外卖推荐

定位附近真实餐饮店，按**营养健康度**或**个人口味偏好**智能推荐，远离"碳水刺客"，吃得更均衡。
纯前端网页，基于高德地图，可一键部署到 GitHub Pages，手机打开即用。

在线体验：`https://moon-yang-cloud.github.io/health_eat/`

---

## 功能

### 健康优选模式
- 从周边真实店铺里，按**健康度 + 距离**排序推荐
- 三种目标：**均衡 / 减脂控碳 / 增肌高蛋白**
- 自动给每家店打"健康分"，高油炸/高碳水的店**标黄预警**并给搭配建议

### 周边美食模式
按需求筛选附近美食：
- **想吃什么**：输入或点标签（火锅、烧烤、日料、川菜、面食、快餐、海鲜、轻食、甜品咖啡…），关键词直接交给高德搜索
- **辣不辣**：不限 / 不吃辣 / 微辣 / 爱吃辣
- **几个人**：1 / 2 / 3-4 / 5 人以上，按人数估算**总价**
- **什么时间**：按当前时间自动判断，或手选早餐 / 午餐 / 晚餐 / 夜宵

### 通用
- 高德精准定位（国内坐标准确），可选 1–5 km 范围
- 真实店名、类型、**距离**、人均/总价预估
- **收藏夹**：点卡片右上角星标收藏喜欢的店，单独一栏随时查看（本地保存）
- **搜索历史**：记录最近定位点，一键回到之前的位置重新搜索
- 跳转：**高德看店/导航**（可靠，可看团购外卖）+ **美团外卖**（复制店名，手机端打开外卖自动定位）

## 配置高德 Key（已配置，换号才需改）

打开 `js/config.js`，修改顶部两行：

```js
const CONFIG = {
  AMAP_KEY: "你的Key",
  AMAP_SECURITY_CODE: "你的安全密钥",
  // ...
};
```

申请：[高德开放平台 lbs.amap.com](https://lbs.amap.com/) → 个人认证 → 创建应用 → 添加 Key，服务平台选 **`Web端(JS API)`**，会得到 Key 和安全密钥两样，两个都要填。

## 测试

用浏览器打开 `tests/test.html`，会自动运行评分引擎与本地存储的单元测试（健康度、价格、辣味/餐次判断、排序、收藏、历史去重等），页面顶部显示"通过 X / X"。

## 部署到 GitHub Pages

**命令行上传（推荐，多文件）**
```bash
git add -A
git commit -m "update"
git push
```
然后到 **Settings → Pages → Source** 选 `Deploy from a branch`，Branch 选 `main` `/ (root)`，**Save**，1-2 分钟后访问 `https://<用户名>.github.io/<仓库名>/`。

**网页上传**
仓库页 → **Add file → Upload files**，把整个项目（含 `index.html`、`css/`、`js/`、`tests/`）拖入并 Commit。

---

## 安全提醒

Key 写在前端会公开。为防盗用，请到高德开放平台给该 Key 的**域名白名单**加上你的 Pages 域名（如 `moon-yang-cloud.github.io`）。

---

## 能做 / 不能做

- 能：真实附近店铺、真实距离、定位、跳转高德
- 不能：每家店的**具体菜品营养与真实菜价**——平台无开放接口，故健康度按品类判断、价格为参考区间
